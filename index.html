How passwords are stored
Passwords are never stored in plain text. Each workspace password is used to encrypt a known sentinel string, which is saved in the browser's localStorage under the key gm_pw2. To verify a password on login, the app decrypts the sentinel and checks it matches. The API key is separately encrypted under the same password and stored under gm_k2. Neither can be read without the correct password. If a client forgets their password, the only option is to use the "Reset workspace" link on the login screen — this clears all their data and lets them start fresh with a new password.
